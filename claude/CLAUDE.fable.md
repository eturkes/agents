# RTK (Rust Token Killer)

CLI proxy auto-applied by the Claude Code hook (`git status` â†’ `rtk git status`, 0-token). Hook rewrites `git status/log/diff/show`, `find`, `ls`, `curl`, `cargo test`, `head`/`tail`â†’`rtk read` (perm-denied â†’ use `Read`); it passes `grep`/`rg`/`sed`/`awk`/`env`/`diff`/`jq`/`sudo` through untouched. Compresses when it has a filter, else passes through. Search = bash `grep`/`rg`. `rtk proxy <cmd>` = raw passthrough â†’ for exact bytes.

## Filters
- **`find`**: `rtk find <dir> -name â€¦` = noise-filtered list; bare path â†’ useless `0 for '<dir>'`; compound (`-not`/`-exec`/â€¦) â†’ hard error + hint â†’ `rtk proxy find`.
- **`rtk format`**: WRITE mode by default (rewrites) â†’ `--check` for read-only, or run the formatter directly (`black`/`ruff`/`prettier --check`; in project venvs â†’ `uv run ruff`).
- **`git diff`/`show`**: condense hunks (leading context lines dropped) â†’ full text via `rtk proxy git â€¦` or redirect; counts via `git rev-list --count`.
- **`json`/`log`** = lossy previews (`json` truncates+reorders, `log` drops INFO detail); auto-`ls` drops dir mtimes â†’ `rtk proxy <cmd>` (or `Read`) for exact.
- **`cargo test`** narrowed (`-p <pkg>` / a test-name filter / `--lib`/`--test`) â†’ one-line summary `cargo test: N passed[, M filtered out] (K suites, Ts)` (per-test lines + `test result:` dropped) â†’ grep/capture of such a run for test NAMES finds nothing (counts ACCURATE â€” trust them, or `rtk proxy cargo test` for per-test detail); a bare `cargo test`/`--workspace` with no narrowing passes raw per-test output through.
- **`rtk: Failed to read file`/`execute command`** = missing file/binary â†’ fix directly.

## Rules
- One `rtk:` error â†’ `rtk proxy` that call. Two failures on one command â†’ cause is elsewhere.
- Filters can degrade silently: `diff`/`show` condensing; build wrappers misreport (`prettier` says "all formatted" on dirty files; `next`/`dotnet` log a failed build as "0 errors"/garbled) â†’ trust exit codes; `rtk proxy` when exact output matters.
- Byte-equality â†’ prove with `cmp`/`sha256sum`; real diffs via `git diff --no-index` or `rtk proxy diff`.
- Filtered `git status` can falsely print `Clean working tree` and return 0 when `-C` names a missing path. Repository/path gates must first test `[ -d "$repo" ]`, then use `rtk proxy git -C "$repo" rev-parse --is-inside-work-tree` and raw status; never use filtered status as existence proof.
- Shell result integrity: capture each exit code immediately (`cmd; rc=$?`) before any `printf`, command substitution, or next command, and label the result; every command overwrites `$?`.

# fff search (tweakcc `rg-fff` â€” shadows `grep`)

CC shell-snapshot repoints `grep`â†’`rg-fff` (argv0 ugrep); **fff-first = DEFAULT** â†’ serves fff **RE2**, relevance-RANKED, with inline markers. Serving needs cwd inside a git repo â€” elsewhere EVERY query defers. `rg`=`/usr/bin/rg` (shadowed only if system rg absent â†’ here untouched). Auto-spawns `rg-fff --daemon <cwd>` per repo-cwd (warm idx, ~30m idle; accumulates, harmless â€” `pgrep -af 'rg[-]fff --daemon'`). Daemon index lags fresh writes: files created/moved seconds ago â†’ silently absent from served hits â†’ `command grep` or re-query for just-written files. RTK passes `grep` thru to fff here; `find` it rewrites â†’ `rtk find`, so fff's find/bfs personality stays dormant (if it ever rewritesâ†’`rtk grep`, fff bypassed too).

- **Output â‰  real grep** â†’ RANKED by relevance over file-order, strips leading `./`, inline markers, lines capped 512B. Clean/order-stable/byte-exact/full-line â†’ `command grep` (=GNU 3.11 BRE, `./`, file-order, untruncated) | `/usr/bin/rg` | `rtk proxy grep`.
- **Zero-match grep â‰  empty (BIG)**: auto fuzzy-fallback â†’ `# rg-fff: 0 EXACTâ€¦` hdr + N ` [~approx]` lines on **STDOUT** (survive `2>/dev/null`, flow into pipes/`$()`), shaped like real `path:line:text`. Truth = **exit 1** + `[~approx]`/leading-`#` (lines appear regardless) â†’ piping/capturing a maybe-zero grep MUST filter `[~approx]`/`#` or use `command grep`. Off: `RG_FFF_NO_FUZZY_FALLBACK=1`.
- **`[def]`** trailing tag = likely definition; rank treats it like any other line (junk/long lines outrank â€” observed def landing last) â†’ trust the tag over position; "read top hit first" misleads here.
- **>512B line** â†’ served truncated + ` [...rg-fff: line truncated at ~512B; Read the file for the full line]` (`-c` counts it once); non-UTF-8 (U+FFFD) long line defers instead â†’ Read file for full line.
- **Served**: literals incl. phrases w/ space/`:`/quote (regex-escaped), RE2 regex (`\b \d \w + * ( ) | {} ?`, empty-capable `x?` OK), `-i` (case-insensitive, any-case pat), `-A/-B/-C N`, `-l`, `-c`, `--include=*.ext`, reldirs + `.` under cwd. No `-r` = top-level only (non-recursive; no "Is a directory" error).
- **DEFERS â†’ embedded ugrep -G = BRE** (clean, file-order, no markers): non-repo cwd (ALL queries), stdin/pipe (`â€¦|grep` ALWAYS, ~52% of greps), single-file arg, `-o`, `-P`/PCRE, regex matching newline (`\s`/negated class), non-ASCII pat, `--no-ignore`/`--include-dir`, abs path, glob â‰  `*.ext`. **Deferred regex = BRE** â†’ RE2/ERE syntax misfires (piped `foo|bar` â†’ matches literal "foo|bar").
- **Dialect when SERVED = RE2** â†’ bare `|`=alt, `+ * ?`=quant, `()`=group, `\d \w \b` classes; `\|` `\+` = LITERAL. Old BRE `\|`=alt / `\+`=1+ advice now valid ONLY for `command grep` / deferred path.
- Knobs: `RG_FFF_FIRST=0` = regexâ†’byte-equiv BRE-mirror (literals STAY fff-ranked+marked); `RG_FFF_LOG=<path>` per-call decision log (`served-daemon`/`fallback-*`); `RG_FFF_DEBUG=1` serve/defer reason on stderr.
